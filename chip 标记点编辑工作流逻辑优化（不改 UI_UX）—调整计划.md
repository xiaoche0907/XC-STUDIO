# chip 标记点编辑工作流逻辑优化（不改 UI/UX）—调整计划

## 目标（不改 UI/UX）

在不改变现有交互与界面呈现的前提下，补齐并优化以下功能逻辑：

- **多标记点按序记忆**：用户依次点 1/2/3… 并输入每个点“怎么改”，模型始终记住顺序与对应指令，不丢、不串。
- **整图一致性生成**：每次生成始终以**整张图片**作为输入，确保风格/光照/结构一致。
- **生成后自动收尾**：生成成功后**标记点自动消失**； **画布与 Agent 区域**同步展示新图；尽量只改标记处及意图相关区域。
- **修复 Gemini Key 过期/未走中转**：确保所有 Gemini 调用强制走中转，避免前端直连导致 Key 报错与不可控。

## 现状问题（已观测）

- 标记点触发编辑后出现报错： **“API Key 无效或已过期…Gemini API Key”**  。
- 现象指向：调用链**疑似绕过中转**，导致 Key 生命周期/权限不可控，用户体验上需要“逐个调整”。

## 设计原则

1. **UI/UX 不变**：所有增强在状态管理、请求编排、后端中转、错误治理、结果回写完成。
2. **状态可恢复、可重试**：任何失败都能根据同一批标记点队列继续，不重复扣费、不重复回写。
3. **一致性优先**：生成以全图为基准；局部修改通过提示词/约束/（可选）遮罩在后端实现，但不新增界面元素。
4. **可观测**：每次生成/回写都有 traceId，可定位“是否走中转”“使用何种策略”“失败原因”。

## 核心逻辑方案（数据与状态）

### 1) 标记点队列（Point Queue）

- 每个标记点记录： `pointId`、 `index(1..n)`、 `geometry(坐标/形状)`、 `instruction(用户输入)`、 `status(待处理/处理中/成功/失败)`。
- 队列归属到： `sessionId`（本次编辑会话）与 `canvasAssetVersionId`（当前画布图版本）。
- **顺序不可变**：生成与回写严格按 index 解释；允许在逻辑层做“合并成一次生成任务”。

### 2) 会话状态（Session State）

- 存储位置：前端仅保留轻量态（用于展示与交互），权威态放服务端（避免刷新/断线丢失）。
- 关键字段：当前基准图版本、点队列快照、最近一次生成任务 ID、生成结果版本链。

## 任务编排（一次生成 vs 多次生成）

- 推荐： **合并为一次生成任务**
  - 输入： `baseImage(整图)` + `pointQueue(1..n)` + `history(必要的上文/风格约束)`。
  - 输出： `newImage` + `editReport(按点回执，可选)`。

- 幂等策略：以 `idempotencyKey = hash(baseImageVersion + pointQueueSnapshot + promptTemplateVersion)` 作为幂等键；重复请求直接返回同一结果。

- 失败重试：
  - 可按任务级重试（同一幂等键）。
  - 可断点：若生成成功但回写失败，可单独重试回写。

## 中转与密钥治理（Gemini 强制走中转）

- 前端不持有长期 Gemini Key；不允许直连 Gemini。

- 服务端中转负责：
  - Key 持有与轮换、过期刷新、配额控制。
  - 统一鉴权与签名（对前端暴露短期 token/签名）。
  - 统一错误码映射（例如 KEY\_EXPIRED/PROXY\_BYPASS/QUOTA 等）与 traceId。

- 校验与防绕过：服务端对来自前端的生成请求做策略校验，确保调用路径只能落到中转。

## 一致性与“只改标记处”的约束

- **全图输入**是前提。
- 约束实现（不改 UI）：
  - 提示词模板：强调保持非标记区域不变，仅修改标记区域相关目标。
  - 必要时服务端引入遮罩（由标记点 geometry 生成）作为模型约束输入；遮罩不需要 UI 展示。
  - 对输出做简单一致性检查（可选）：如非标记区域结构差异过大则自动降级重试或提示失败。

## 生成后收尾与回写

- 成功后执行：
  1. 将 `newImage` 写入资产系统，生成新 `canvasAssetVersionId`。
  2. **清空标记点队列**（或将其归档为已完成快照）。
  3. 推送/拉取更新：画布与 Agent 区域同步展示最新版本。

- 版本管理：保留最近 N 个版本支持回滚（不新增 UI 的情况下可在现有机制里复用）。

## 错误处理与提示（不改 UI）

- 仍使用现有提示位，但信息更可诊断：
  - 统一错误码 + traceId + 建议动作（重试/稍后/联系管理员）。

- 常见错误覆盖：Key 过期、配额、网络、模型拒答、回写失败、版本冲突。

## 验收标准（关键用例）

1. 多标记点：连续添加 3+ 点，跨多轮输入后仍保持 1/2/3 对应关系不丢不串。
2. 生成一致性：同一张图多次编辑风格稳定；输出以整图为基准。
3. 局部修改：非标记区域变化可控（按阈值/肉眼验收），主要修改集中在标记意图区域。
4. 生成后收尾：成功后标记点自动消失；画布与 Agent 展示同步更新到新图。
5. 中转生效：不再出现“Gemini Key 过期/无效”的前端直连错误；日志可证明调用走中转。
6. 稳定性：重试不重复扣费、不重复回写；断线/刷新可恢复到同一会话状态。

## 里程碑建议（不涉及 UI 变更）

- M1：调用链梳理 + 中转强制接入 + 关键日志/traceId
- M2：标记点队列/会话状态落库 + 幂等/重试
- M3：全图生成编排 + 只改标记处约束（提示词/可选遮罩）
- M4：回写与版本链 + 自动清空标记点
- M5：联调验收 + 性能/成本评估与参数固化
